# 需求文档

## 简介

本文档规定了增强 DataConfig.vue 组件的计算指标配置能力的需求。该功能使用户能够定义聚合指标(SUM、AVG、COUNT、MAX、MIN)和计算指标(条件比率、简单比率、条件求和、自定义表达式),用于金融指标计算,如不良贷款率和资本充足率。

## 术语表

- **数据配置组件(DataConfig_Component)**: 负责为仪表板可视化配置数据源和指标的 Vue.js 组件
- **基础指标(Base_Metric)**: 可以使用标准 SQL 函数(SUM、AVG、COUNT、MAX、MIN)进行聚合的数据集基本字段
- **计算指标(Computed_Metric)**: 使用数学表达式或条件逻辑从基础指标计算得出的派生指标
- **指标转换器(Metric_Converter)**: 将指标配置转换为 SQL 查询的后端服务
- **SQL验证器(SQL_Validator)**: 验证 SQL 表达式以防止注入攻击的安全组件
- **SQL方言适配器(SQL_Dialect_Adapter)**: 处理特定数据库 SQL 语法差异的组件
- **指标配置DTO(Metric_Configuration_DTO)**: 包含指标定义和计算规则的数据传输对象
- **表达式解析器(Expression_Parser)**: 解析和验证自定义数学表达式的组件

## 需求

### 需求 1: 基础指标聚合

**用户故事:** 作为仪表板设计者,我希望从数据集字段配置聚合指标,以便在可视化中显示汇总数据。

#### 验收标准

1. WHEN 用户从数据集中选择数值字段时, THE 数据配置组件 SHALL 显示可用的聚合函数(SUM、AVG、COUNT、MAX、MIN)
2. WHEN 用户选择聚合函数时, THE 数据配置组件 SHALL 创建包含字段名和聚合类型的基础指标配置
3. WHEN 用户为指标提供自定义别名时, THE 数据配置组件 SHALL 存储别名并在可视化中使用它
4. THE 指标转换器 SHALL 为所有支持的聚合类型生成有效的 SQL 聚合表达式
5. WHEN 配置多个基础指标时, THE 数据配置组件 SHALL 在可管理的列表中显示它们,并提供编辑和删除功能

### 需求 2: 条件比率指标

**用户故事:** 作为金融分析师,我希望计算条件比率(例如不良贷款率),以便监控风险指标。

#### 验收标准

1. WHEN 用户选择"条件比率"指标类型时, THE 数据配置组件 SHALL 显示分子条件、分母条件和基础字段的配置字段
2. WHEN 用户定义分子条件(例如 status='NPL')时, THE SQL验证器 SHALL 在接受之前验证条件语法
3. WHEN 用户定义分母条件时, THE SQL验证器 SHALL 在接受之前验证条件语法
4. THE 指标转换器 SHALL 使用 CASE 语句生成 SQL 表达式: `SUM(CASE WHEN <numerator_condition> THEN <field> ELSE 0 END) / NULLIF(SUM(CASE WHEN <denominator_condition> THEN <field> ELSE 0 END), 0)`
5. WHEN 分母计算结果为零时, THE 生成的 SQL SHALL 返回 NULL 以防止除零错误

### 需求 3: 简单比率指标

**用户故事:** 作为仪表板设计者,我希望计算两个指标之间的简单比率,以便显示基于百分比的指标。

#### 验收标准

1. WHEN 用户选择"简单比率"指标类型时, THE 数据配置组件 SHALL 显示下拉菜单以从已配置的基础指标中选择分子和分母指标
2. WHEN 用户选择分子和分母指标时, THE 数据配置组件 SHALL 验证两个指标都存在
3. THE 指标转换器 SHALL 生成 SQL 表达式: `<numerator_metric> / NULLIF(<denominator_metric>, 0)`
4. WHEN 分母指标计算结果为零时, THE 生成的 SQL SHALL 返回 NULL
5. WHEN 用户启用百分比显示时, THE 数据配置组件 SHALL 在可视化层将结果乘以 100

### 需求 4: 条件求和指标

**用户故事:** 作为金融分析师,我希望计算条件求和(例如不良贷款总额),以便根据特定标准聚合数据。

#### 验收标准

1. WHEN 用户选择"条件求和"指标类型时, THE 数据配置组件 SHALL 显示用于选择基础字段和定义条件的字段
2. WHEN 用户定义条件时, THE SQL验证器 SHALL 验证条件语法并拒绝无效表达式
3. THE 指标转换器 SHALL 生成 SQL 表达式: `SUM(CASE WHEN <condition> THEN <field> ELSE 0 END)`
4. WHEN 条件包含用户输入时, THE SQL验证器 SHALL 清理输入以防止 SQL 注入
5. THE 数据配置组件 SHALL 支持在同一字段上使用不同条件的多个条件求和指标

### 需求 5: 自定义表达式指标

**用户故事:** 作为高级用户,我希望定义组合多个指标的自定义数学表达式,以便创建复杂的计算指标。

#### 验收标准

1. WHEN 用户选择"自定义表达式"指标类型时, THE 数据配置组件 SHALL 显示用于输入数学表达式的文本输入框
2. WHEN 用户输入表达式时, THE 表达式解析器 SHALL 验证语法并识别引用的指标
3. THE 表达式解析器 SHALL 支持基本算术运算符(+、-、*、/)和用于优先级的括号
4. WHEN 表达式引用指标时, THE 表达式解析器 SHALL 验证引用的指标在配置中存在
5. THE 指标转换器 SHALL 通过将指标引用替换为相应的 SQL 表达式来生成 SQL 表达式
6. WHEN 表达式包含除法时, THE 指标转换器 SHALL 使用 NULLIF 包装分母以防止除零
7. THE SQL验证器 SHALL 拒绝包含可能启用 SQL 注入的 SQL 关键字、函数或特殊字符的表达式

### 需求 6: SQL 注入防护

**用户故事:** 作为系统管理员,我希望所有用户提供的指标配置都经过验证和清理,以便系统免受 SQL 注入攻击。

#### 验收标准

1. WHEN 用户提交任何指标配置时, THE SQL验证器 SHALL 在处理之前验证所有用户提供的字符串
2. THE SQL验证器 SHALL 拒绝包含 SQL 关键字(SELECT、INSERT、UPDATE、DELETE、DROP、UNION 等)的配置
3. THE SQL验证器 SHALL 拒绝包含 SQL 注释语法(-- 或 /* */)的配置
4. THE SQL验证器 SHALL 拒绝包含分号或其他语句终止符的配置
5. WHEN 验证失败时, THE 数据配置组件 SHALL 显示清晰的错误消息,指示安全违规
6. THE 指标转换器 SHALL 对所有动态值使用参数化查询或适当的转义
7. THE SQL验证器 SHALL 维护允许的字段名白名单,并拒绝对不存在字段的引用

### 需求 7: 数据库方言支持

**用户故事:** 作为系统架构师,我希望指标配置能够在不同的数据库系统中工作,以便平台支持多个数据源。

#### 验收标准

1. WHEN 为指标生成 SQL 时, THE SQL方言适配器 SHALL 从数据源配置中检测目标数据库类型
2. THE SQL方言适配器 SHALL 为 MySQL 数据源生成 MySQL 兼容的 SQL
3. THE SQL方言适配器 SHALL 为 PostgreSQL 数据源生成 PostgreSQL 兼容的 SQL
4. THE SQL方言适配器 SHALL 为 ClickHouse 数据源生成 ClickHouse 兼容的 SQL
5. WHEN 需要特定于数据库的函数时, THE SQL方言适配器 SHALL 为目标数据库使用适当的语法
6. WHEN 指标配置使用目标数据库不支持的功能时, THE SQL方言适配器 SHALL 返回清晰的错误消息

### 需求 8: 性能保护

**用户故事:** 作为系统管理员,我希望指标计算具有性能保护措施,以便复杂查询不会影响系统稳定性。

#### 验收标准

1. WHEN 用户在单个数据集中配置超过 20 个指标时, THE 数据配置组件 SHALL 显示有关潜在性能影响的警告
2. WHEN 自定义表达式超过 500 个字符时, THE 表达式解析器 SHALL 拒绝它并显示错误消息
3. WHEN 指标配置将生成超过 10,000 个字符的 SQL 查询时, THE 指标转换器 SHALL 拒绝它
4. THE 数据配置组件 SHALL 将计算指标的嵌套深度限制为 3 层
5. WHEN 查询执行时间超过 30 秒时, THE 后端 SHALL 超时并返回错误消息

### 需求 9: 指标配置持久化

**用户故事:** 作为仪表板设计者,我希望我的指标配置与数据集一起保存,以便我可以在多个可视化中重用它们。

#### 验收标准

1. WHEN 用户保存带有指标配置的数据集时, THE 后端 SHALL 将指标配置DTO序列化为 JSON
2. THE 后端 SHALL 将 JSON 配置存储在数据集的配置字段中
3. WHEN 用户加载数据集时, THE 后端 SHALL 反序列化 JSON 配置并将其返回给前端
4. WHEN 数据集被删除时, THE 后端 SHALL 删除所有关联的指标配置
5. THE 后端 SHALL 在使用反序列化的配置生成 SQL 查询之前验证它

### 需求 10: 指标管理用户界面

**用户故事:** 作为仪表板设计者,我希望有一个直观的界面来管理指标,以便我可以高效地配置复杂的计算。

#### 验收标准

1. THE 数据配置组件 SHALL 显示带有"基础指标"和"计算指标"部分的选项卡界面
2. WHEN 用户点击"添加指标"时, THE 数据配置组件 SHALL 显示带有指标类型选择的模态对话框
3. WHEN 用户选择指标类型时, THE 数据配置组件 SHALL 显示特定于类型的配置字段
4. THE 数据配置组件 SHALL 为每个指标显示生成的 SQL 表达式的预览
5. WHEN 用户将鼠标悬停在指标上时, THE 数据配置组件 SHALL 显示带有指标计算逻辑的工具提示
6. THE 数据配置组件 SHALL 支持拖放重新排序列表中的指标
7. WHEN 用户删除基础指标时, THE 数据配置组件 SHALL 在计算指标依赖它时发出警告

### 需求 11: 指标验证和错误处理

**用户故事:** 作为仪表板设计者,我希望在配置指标时获得清晰的验证反馈,以便我可以快速识别和修复配置错误。

#### 验收标准

1. WHEN 用户输入无效的字段名时, THE 数据配置组件 SHALL 显示内联错误消息
2. WHEN 用户输入无效的条件语法时, THE 数据配置组件 SHALL 显示具体的语法错误
3. WHEN 用户在表达式中引用不存在的指标时, THE 数据配置组件 SHALL 突出显示无效引用
4. WHEN 后端验证失败时, THE 数据配置组件 SHALL 显示服务器端错误消息
5. THE 数据配置组件 SHALL 防止保存带有验证错误的配置
6. WHEN 指标配置有效时, THE 数据配置组件 SHALL 显示成功指示器

### 需求 12: 指标测试和预览

**用户故事:** 作为仪表板设计者,我希望在保存之前测试指标计算,以便我可以验证结果是否正确。

#### 验收标准

1. WHEN 用户点击"测试指标"时, THE 数据配置组件 SHALL 将指标配置发送到后端
2. THE 后端 SHALL 使用 LIMIT 子句执行生成的 SQL 查询以检索样本结果
3. THE 数据配置组件 SHALL 以表格格式显示样本结果
4. WHEN 查询执行失败时, THE 数据配置组件 SHALL 显示错误消息和 SQL 查询以进行调试
5. THE 数据配置组件 SHALL 显示测试查询的执行时间
