## 注意事项

### 安全优先

- 所有用户输入必须经过SQLValidator验证
- 测试必须包含SQL注入防护验证
- 错误消息不能泄露SQL结构信息

### 性能保护

- 指标数量限制: 20个(软限制,显示警告)
- 表达式长度限制: 500字符
- SQL查询长度限制: 10,000字符
- 嵌套深度限制: 3层
- 查询超时: 30秒

### 测试要求

- 每个属性测试最少100次迭代
- 使用jqwik(后端)和fast-check(前端)
- 标签格式: `Feature: enhanced-data-config, Property N: <property_text>`
- 单元测试覆盖边界情况和错误条件

### 数据库兼容性

- 必须支持: MySQL, PostgreSQL, ClickHouse, Doris
- 测试每种数据库的SQL生成
- 处理不支持功能的情况

## 实施顺序说明

任务按照以下原则组织:

1. **安全优先**: 首先实现安全组件(SQLValidator, ExpressionParser)
2. **由内而外**: 从核心逻辑(SQL生成)到API再到UI
3. **增量验证**: 每个阶段结束都有checkpoint
4. **测试驱动**: 每个核心组件都有对应的属性测试和单元测试

标记为 `*` 的任务为可选测试任务,可以根据项目进度决定是否实施。
