# 指标血缘功能使用指南

## 📖 功能概述

指标血缘功能允许用户在看板中点击图表，查看：
- **业务口径**：指标的业务定义和计算逻辑
- **技术血缘**：指标的上下游依赖关系（可视化图谱）
- **数据查询**：指标的实际数据，支持筛选和导出

---

## 🚀 快速开始

### 第一步：准备测试数据

1. **执行测试数据脚本**

```bash
# 在项目根目录执行
mysql -u root -p iras < INSERT_TEST_DATA.sql
```

该脚本会创建3个测试指标及其血缘关系：
- `TEST_001` 原销售额（基础指标）
- `TEST_002` 月度销售额（聚合指标）
- `TEST_003` 销售达成率（衍生指标）

血缘链路：
```
原销售额 → [聚合] → 月度销售额 → [计算] → 销售达成率
```

2. **验证数据是否创建成功**

```sql
SELECT id, metric_code, metric_name, business_definition
FROM bi_metric_metadata
WHERE metric_code LIKE 'TEST_%';
```

### 第二步：为图表配置指标ID

**方式1：通过界面配置（推荐）**

1. 进入看板设计器：`/bi/dashboard/designer/:id`
2. 点击某个图表组件
3. 在"数据配置"中找到"指标ID"字段
4. 输入指标ID（如：从第一步查询得到的ID）
5. 保存配置

**方式2：直接修改数据库**

```sql
-- 为图表组件关联指标（假设图表组件ID=123，指标ID=1）
UPDATE bi_dashboard_component
SET data_config = JSON_SET(
    COALESCE(data_config, '{}'),
    '$.metricId',
    1
)
WHERE id = 123;
```

**方式3：通过API配置**

```javascript
// 调用看板组件更新API
PUT /bi/dashboard/component/123
{
  "dataConfig": {
    "datasetId": 1,
    "metricId": 1,  // ← 关键：配置指标ID
    "dimensions": [...],
    "metrics": [...]
  }
}
```

### 第三步：查看指标详情

1. **进入看板查看页面**
   - 访问：`http://localhost:80/views/bi/dashboard/view/1`

2. **识别可点击的图表**
   - 已关联指标的图表会有**蓝色边框**高亮（鼠标悬停时）
   - 光标变为手型指针

3. **点击图表**
   - 自动打开指标详情对话框
   - 默认显示"指标规范"标签页

## 📋 功能详解

### 标签页1：指标规范

**显示内容：**
- 指标编码、名称、状态
- 责任部门、数据新鲜度、更新频率
- 业务定义（可编辑）
- 技术公式
- 计算逻辑说明
- 创建/更新信息

**操作：**
- 点击"编辑"按钮修改业务定义、技术公式、计算逻辑
- 修改后自动同步到数据库

---

### 标签页2：血缘图谱

**可视化展示：**

```
┌─────────────────────────────────────────┐
│  [完整图谱] [上游血缘] [下游血缘]      │
├─────────────────────────────────────────┤
│                                         │
│   [数据源] → [数据集] → [原销售额]     │
│                           ↓            │
│                      [月度销售额]       │
│                           ↓            │
│                      [销售达成率]       │
│                                         │
│  • 蓝色节点：指标                        │
│  • 绿色节点：数据集                      │
│  • 橙色节点：数据源                      │
│  • 箭头标签：转换类型                    │
└─────────────────────────────────────────┘
```

**节点类型：**
- 🔵 **指标** (metric)：蓝色
- 🟢 **数据集** (dataset)：绿色
- 🟠 **数据源** (datasource)：橙色
- 🟣 **数据表** (table)：紫色
- 🔴 **字段** (field)：红色

**转换类型：**
- **aggregation**：聚合（求和、计数等）
- **calculation**：计算（加减乘除）
- **filter**：过滤（条件筛选）
- **derived_from**：衍生
- **depends_on**：依赖

**操作：**
- 点击节点查看详细信息
- 切换查看模式（完整/上游/下游）
- 放大、缩小图谱
- 导出图谱为PNG图片

---

### 标签页3：数据查询

**查询功能：**
- 时间范围筛选
- 排序字段和方式
- 数据源/数据集筛选
- 自定义过滤条件

**数据展示：**
- 数据概览卡片（总记录数、新鲜度、更新频率）
- 数据表格（动态列生成）
- 分页浏览

**操作：**
- 添加自定义过滤条件
- 执行查询
- 导出数据（Excel格式）
- 刷新缓存

## 🎯 典型使用场景

### 场景1：数据分析师查看指标口径

**问题：** "这个销售额指标是怎么算的？"

**操作：**
1. 打开包含销售额图表的看板
2. 点击销售额图表
3. 查看"指标规范"标签页
4. 阅读"业务定义"和"计算逻辑"

**结果：** 清楚了解指标的业务口径和技术实现

---

### 场景2：追溯数据来源

**问题：** "这个指标的原始数据从哪来？"

**操作：**
1. 点击指标图表
2. 切换到"血缘图谱"标签页
3. 选择"上游血缘"模式
4. 查看蓝色箭头指向的源头节点
5. 点击节点查看详情

**结果：** 清晰展示从数据源→数据集→指标的全链路

---

### 场景3：影响分析

**问题：** "如果修改这个指标，会影响哪些下游指标？"

**操作：**
1. 点击指标图表
2. 切换到"血缘图谱"标签页
3. 选择"下游血缘"模式
4. 查看所有依赖此指标的下游节点

**结果：** 快速识别所有受影响的指标和报表

---

### 场景4：数据异常排查

**问题：** "这个指标的数据对不对？"

**操作：**
1. 点击指标图表
2. 切换到"数据查询"标签页
3. 设置时间范围和过滤条件
4. 查看原始数据
5. 导出数据进行验证

**结果：** 快速定位数据异常的根本原因

## ⚙️ 权限控制

所有功能都支持权限控制：

| 功能 | 权限标识 |
|------|----------|
| 查看指标规范 | `bi:metadata:query` |
| 编辑指标规范 | `bi:metadata:edit` |
| 查看血缘图谱 | `bi:lineage:query` |
| 查询指标数据 | `bi:metric:data:query` |
| 导出指标数据 | `bi:metric:data:export` |

在 Vue 组件中使用：
```vue
<el-button v-hasPermi="['bi:metadata:edit']">编辑</el-button>
```

## 🔧 配置说明

### 图表组件配置

**dataConfig 结构：**
```json
{
  "datasetId": 1,        // 数据集ID
  "metricId": 1,         // ← 指标ID（关键配置）
  "dimensions": [
    {
      "fieldName": "order_date",
      "comment": "订单日期"
    }
  ],
  "metrics": [
    {
      "fieldName": "sales_amount",
      "comment": "销售额"
    }
  ],
  "filters": [...]
}
```

**关键配置项：**
- `metricId`：指标ID，配置后图表可点击查看详情
- `datasetId`：数据集ID，用于数据查询

## 🐛 常见问题

### Q1: 点击图表没有反应？

**检查：**
1. 图表是否配置了 `metricId`
2. 是否在查看模式（非编辑模式）
3. 浏览器控制台是否有错误

**验证 SQL：**
```sql
SELECT id, component_name, data_config
FROM bi_dashboard_component
WHERE data_config LIKE '%"metricId"%';
```

### Q2: 血缘图谱显示为空？

**检查：**
1. 数据库中是否有血缘关系数据
2. API 接口是否正常返回

**验证 SQL：**
```sql
SELECT COUNT(*) as lineage_count
FROM bi_metric_lineage;
```

### Q3: 数据查询报错？

**检查：**
1. 数据集ID是否正确
2. 数据源连接是否正常
3. 是否有查询权限
