# 不良贷款率配置指南

## 业务需求
配置某机构某日的不良贷款率指标

## 数据源
- 表名: `t_cbis_t_daikuan_hzwdb`
- 关键字段:
  - `jkye`: 借款余额（decimal 20,2）
  - `wjfl`: 五级分类（varchar）- 值包括：正常、关注、次级、可疑、损失
  - `sjbsjgid`: 数据报送机构ID（varchar）
  - `load_date`: 数据加载日期（varchar）

## 配置方案

### 方案一：使用基础指标 + 简单比率（推荐）

#### 步骤1: 在DataConfig中配置过滤条件
在"数据源配置"选项卡的"过滤条件"部分添加：

1. **机构筛选**
   - 字段: `sjbsjgid`
   - 操作符: `=`
   - 值: `具体机构ID`（如：'001'）

2. **日期筛选**
   - 字段: `load_date`
   - 操作符: `=`
   - 值: `具体日期`（如：'2024-01-15'）

#### 步骤2: 配置基础指标1 - 不良贷款余额
在"基础指标"选项卡点击"添加基础指标"：

- **指标名称**: `npl_balance`
- **指标别名**: `不良贷款余额`
- **字段**: `jkye`
- **聚合函数**: `SUM`
- **过滤条件**: `wjfl IN ('次级', '可疑', '损失')`

生成的SQL:
```sql
SUM(jkye) FILTER (WHERE wjfl IN ('次级', '可疑', '损失')) AS npl_balance
```

#### 步骤3: 配置基础指标2 - 贷款余额总计
继续添加基础指标：

- **指标名称**: `total_balance`
- **指标别名**: `贷款余额总计`
- **字段**: `jkye`
- **聚合函数**: `SUM`
- **过滤条件**: (留空)

生成的SQL:
```sql
SUM(jkye) AS total_balance
```

#### 步骤4: 配置计算指标 - 不良贷款率
在"计算指标"选项卡点击"添加计算指标"：

- **指标名称**: `npl_ratio`
- **指标别名**: `不良贷款率`
- **计算类型**: `简单比率`
- **分子指标**: `npl_balance`
- **分母指标**: `total_balance`
- **显示为百分比**: ✓ 是

生成的SQL:
```sql
npl_balance / NULLIF(total_balance, 0) AS npl_ratio
```

#### 最终生成的完整SQL
```sql
SELECT 
  SUM(CASE WHEN wjfl IN ('次级', '可疑', '损失') THEN jkye ELSE 0 END) AS npl_balance,
  SUM(jkye) AS total_balance,
  SUM(CASE WHEN wjfl IN ('次级', '可疑', '损失') THEN jkye ELSE 0 END) / 
    NULLIF(SUM(jkye), 0) * 100 AS npl_ratio
FROM t_cbis_t_daikuan_hzwdb
WHERE sjbsjgid = '001' 
  AND load_date = '2024-01-15'
```

### 方案二：使用条件比率（备选）

#### 步骤1: 配置过滤条件
同方案一，配置机构和日期筛选

#### 步骤2: 配置计算指标 - 不良贷款率
在"计算指标"选项卡点击"添加计算指标"：

- **指标名称**: `npl_ratio`
- **指标别名**: `不良贷款率`
- **计算类型**: `条件比率`
- **字段**: `jkye`
- **分子条件**: `wjfl IN ('次级', '可疑', '损失')`
- **分母条件**: `1=1` 或留空（表示所有记录）
- **显示为百分比**: ✓ 是

生成的SQL:
```sql
SELECT 
  SUM(CASE WHEN wjfl IN ('次级', '可疑', '损失') THEN jkye ELSE 0 END) / 
    NULLIF(SUM(CASE WHEN 1=1 THEN jkye ELSE 0 END), 0) * 100 AS npl_ratio
FROM t_cbis_t_daikuan_hzwdb
WHERE sjbsjgid = '001' 
  AND load_date = '2024-01-15'
```

## 当前功能的限制

### ✅ 支持的功能
1. ✅ 基础指标的过滤条件（WHERE子句）
2. ✅ 条件比率计算
3. ✅ 简单比率计算
4. ✅ 全局过滤条件（应用到整个查询）
5. ✅ 百分比显示

### ⚠️ 存在的限制
1. ⚠️ 基础指标的过滤条件中使用 `IN` 子句需要验证SQL安全性
2. ⚠️ 机构和日期筛选必须在全局过滤条件中配置，不能在指标级别配置
3. ⚠️ 如果需要对比多个机构或多个日期，需要使用维度字段而不是过滤条件

### ❌ 不支持的功能
1. ❌ 指标级别的维度筛选（如：只对某个指标应用机构筛选）
2. ❌ 动态参数化的过滤条件（如：用户在仪表板上选择机构和日期）

## 解决限制的方案

### 限制1: IN子句的SQL安全性
**问题**: 基础指标的过滤条件中使用 `IN ('次级', '可疑', '损失')` 可能被SQL验证器拒绝

**解决方案**:
1. 检查 `SQLValidator.validateCondition()` 是否支持 `IN` 子句
2. 如果不支持，使用 `OR` 替代：`wjfl = '次级' OR wjfl = '可疑' OR wjfl = '损失'`

### 限制2: 动态参数化筛选
**问题**: 用户希望在仪表板上动态选择机构和日期

**解决方案**:
1. 使用查询条件组件（QueryConditionConfig）
2. 配置机构和日期为查询参数
3. 在图表查询时动态注入参数值

**配置步骤**:
1. 在仪表板设计器中添加查询条件组件
2. 配置两个查询条件：
   - 机构选择器（下拉框，绑定到 `sjbsjgid` 字段）
   - 日期选择器（日期选择器，绑定到 `load_date` 字段）
3. 在图表组件的数据配置中，不配置固定的过滤条件
4. 通过字段联动功能，将查询条件的值传递给图表

### 限制3: 多机构对比
**问题**: 需要对比多个机构的不良贷款率

**解决方案**:
1. 不使用过滤条件筛选机构
2. 将 `sjbsjgid` 作为维度字段拖入"维度"区域
3. 使用柱状图或折线图展示多个机构的对比

**配置调整**:
- 维度: `sjbsjgid` (机构ID)
- 指标: `npl_ratio` (不良贷款率)
- 过滤条件: 只保留日期筛选 `load_date = '2024-01-15'`

## 常见问题

### Q1: 过滤条件中的 IN 子句报错怎么办？
A: 改用 OR 连接：`wjfl = '次级' OR wjfl = '可疑' OR wjfl = '损失'`

### Q2: 如何配置多个日期的趋势分析？
A: 
1. 不在过滤条件中配置日期
2. 将 `load_date` 作为维度字段
3. 使用折线图展示趋势

### Q3: 如何让用户在仪表板上选择机构和日期？
A: 使用查询条件组件（QueryConditionConfig）配置动态参数

### Q4: 不良贷款率显示为小数而不是百分比？
A: 在计算指标配置中勾选"显示为百分比"选项

## 总结

**当前功能可以配置成功**，推荐使用方案一（基础指标 + 简单比率）：
- ✅ 配置清晰，易于维护
- ✅ 基础指标可复用
- ✅ 支持过滤条件
- ⚠️ 需要验证 IN 子句的SQL安全性
- ⚠️ 动态参数需要配合查询条件组件使用

如果遇到SQL验证问题，可以使用方案二（条件比率）或将 IN 改为 OR 连接。
