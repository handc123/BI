# 2026-03-10 穿透明细规则编辑重构设计（A/B分离 + 排序 + 日期动态控件）

## 1. 目标
- 区块A从只读标签升级为可编辑条件行：字段只读、操作符和值可编辑。
- 区块B继续保留“条件组 + 组间 AND/OR”能力。
- 新增排序规则 `sortRules`，支持多条排序。
- 日期字段根据操作符动态切换单日/区间控件。

## 2. 已确认决策
- A/B 分离展示（A在上，B在下）。
- 区块A条件来源于图表带入条件。
- 区块A字段只读不可改；可改操作符和值。
- 区块A编辑结果参与查询并覆盖同字段原始带入条件。
- 区块B继续作为高级规则补充/覆盖。
- 新增排序数组：`sortRules: [{ field, order }]`，支持“+”新增多条。
- 日期比较按“日期粒度”处理（自动补全当天起止时间语义）。

## 3. 页面结构
### 3.1 区块A：带入条件（可编辑）
每条筛选项结构：
- 标签：`* 筛选规则：字段中文名`（必填显示红色 `*`）
- 操作符下拉：等于/包含/介于/为空/非空/大于/小于等
- 值控件：输入框/选择器/多选标签

字段锁定规则：
- `field` 只读，不可修改，不可删除
- `operator/value` 可编辑

### 3.2 排序区（置于区块A内）
- 行结构：字段选择 + 升序/降序
- 支持“+”添加更多排序条件
- 支持删除排序行

### 3.3 区块B：用户规则组
- 保持现有交互与数据结构
- 支持条件组增删、组间 AND/OR、组内条件增删

## 4. 日期字段动态控件
按操作符动态切换：
1. `=`：单日日期选择器
2. `BETWEEN`：日期区间选择器
3. `>`/`>=`/`<`/`<=`：单日日期选择器
4. `IS NULL`/`IS NOT NULL`：隐藏/禁用值输入

日期语义：
- 后端比较按日期粒度处理（与现有 `QueryExecutor` 日期补全规则一致）

## 5. 查询合并规则
1. 从图表带入条件生成区块A初始行
2. 用户编辑后形成 `inheritedConditions`
3. 区块B生成 `ruleGroups`
4. 后端执行时维持“同字段用户规则优先覆盖”
5. 应用 `sortRules`（字段白名单 + 升降序白名单）

## 6. 接口协议调整
- `POST /bi/drill/query`
- `POST /bi/drill/queryByField`

新增可选参数：
- `sortRules`: `[{ field: string, order: 'ASC'|'DESC' }]`

兼容性：
- 未传 `sortRules` 时保持旧行为不变

## 7. 校验与提示
- `BETWEEN`：必须2个值
- `IN/NOT IN`：至少1个值
- `IS NULL/IS NOT NULL`：值为空
- 排序行 `field/order` 成对必填
- 非法字段/操作符拒绝并提示

## 8. 验收标准
1. 区块A字段只读，操作符和值可编辑并生效
2. 区块B组规则功能不回归
3. A编辑后查询生效，B同字段覆盖A
4. 支持多条排序并生效
5. 日期字段根据操作符动态切换单日/区间控件
6. 计算字段链路与原始字段链路均支持 A/B + 排序
