# 数据集行级权限配置回退设计

**目标**
- 完整回退“数据集行级权限配置”前后端改动，恢复到本分支 `HEAD` 基线状态。
- 不影响当前仓库其他业务改动。

## 范围
- 前端：数据集页面新增的“行级权限配置”UI和调用链。
- 后端：行级权限控制器、服务、Mapper、SQL 脚本与相关接入逻辑。

## 非目标
- 不重构数据权限体系。
- 不改动与行级权限无关的模块。

## 方案
1. 先通过 `git status` + `git diff --name-only` 识别“行级权限”相关文件列表。
2. 对已跟踪文件执行 `git restore -- <files>` 回退到 `HEAD`。
3. 对新增未跟踪文件执行删除（仅限行级权限相关文件）。
4. 做最小验证：
   - 校验关键文件不再包含行级权限入口；
   - 校验前端页面无“数据集行级权限配置”区块。

## 风险与规避
- 风险：工作区很脏，误回退无关改动。
- 规避：仅对白名单文件操作，先列清单后执行，执行后再次核对变更集。

## 验收标准
- `ui/src/views/bi/dataset/index.vue` 不再包含行级权限 UI 与 API 调用。
- `ui/src/api/bi/dataset.js` 不再包含行级权限接口方法。
- 后端新增的 `DatasetPermissionConfig*` 文件被回退/移除。
- 查询引擎不再引用行级权限配置入口。
