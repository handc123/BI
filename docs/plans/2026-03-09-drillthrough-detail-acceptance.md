# 2026-03-09 BI穿透明细验收记录

## 1. 验收范围
- 图表点击跳转穿透明细页
- 区块A已带入条件展示（图表查询条件 + 指标口径条件）
- 区块B多条件组（AND/OR）动态查询
- 区块C明细结果分页查询
- 后端条件合并规则（用户规则覆盖带入条件）

## 2. 验收清单
1. 单指标图表点击后进入 `/bi/dashboard/drill/:dashboardId`，URL包含 `metricId`
2. 多指标图表点击不同折线，进入页面的 `metricId` 与点击指标一致
3. 区块A显示图表查询条件（如机构、日期、查询组件快照）
4. 区块A显示指标口径解析条件；解析失败提示不阻断查询
5. 点击“转为可编辑条件”后，区块B新增对应条件行
6. 区块B支持新增/删除条件组，组间关系支持 AND/OR
7. 区块B支持新增/删除条件行，字段/操作符中文展示
8. 点击查询后区块C返回明细表格并支持分页
9. 同字段冲突时，区块B用户规则覆盖区块A带入条件
10. `NOT IN / IS NULL / IS NOT NULL` 等操作符在后端可正确生效（不丢条件）
11. 图表点击不再弹出指标详情，交互统一为“单击跳穿透”

## 3. 后端关键行为确认
- 字段白名单校验：仅允许数据集字段参与穿透查询
- 操作符归一化：支持符号与中文别名，统一转换后处理
- 条件执行策略：
  - 可下推操作符（`EQ/NE/GT/GTE/LT/LTE/LIKE/IN/BETWEEN`）先下推查询引擎
  - 不可下推操作符（如 `NOT IN/IS NULL/IS NOT NULL`）在内存结果上补过滤
- 合并优先级：用户规则字段优先，覆盖同字段带入条件

## 4. 已知限制
1. 目前区块B单值输入框对 `IN/BETWEEN` 使用逗号分隔文本，尚未提供多值选择UI
2. 后端规则组按 left-associative 计算：`g1 (rel2 g2) (rel3 g3)...`
3. `LIKE` 目前为包含匹配，不区分大小写策略取决于底层数据库/排序规则
4. 受查询引擎能力限制，部分复杂操作符采用“先查后滤”，超大数据量下性能受影响
5. 指标详情弹窗已下线，若需要指标治理信息需从指标管理页面查看
