# 2026-03-10 机构级指标解析与计算字段元数据化穿透设计

## 1. 目标
- 保持穿透链路以 `metricId` 为唯一驱动。
- 图表中的计算字段必须先注册为指标元数据，禁止仅图表临时定义直接穿透。
- 适配“不同机构指标体系独立（metricCode 可重复）”的权限模型。

## 2. 已确认决策
- 计算字段穿透前置条件：必须在本机构指标元数据存在。
- 指标体系：按机构独立维护，`metricCode` 非全局唯一。
- 运行时匹配方式：按“当前登录机构 + 指标标识”解析出机构内 `metricId`。

## 3. 主流BI对齐原则
- 指标治理优先于图表临时表达式：图表引用已治理指标，而不是直接依赖临时 SQL 表达式。
- 多租户/多组织下，指标主键应是机构域内解析结果，不应跨机构复用同一 `metricId`。
- 前端可传逻辑标识（如 `metricCode`），后端在权限上下文中解析物理 `metricId`。

## 4. 数据模型与约束
### 4.1 指标唯一性
- 建议唯一约束从全局 `metric_code` 调整为 `(org_id, metric_code)`。
- 同一机构内 `metricCode` 唯一；不同机构可重复。

### 4.2 图表配置字段
- 图表指标配置中必须持久化逻辑标识（建议 `metricCode`），且标注是否计算字段来源。
- 禁止只保存 `calculatedFields.expression` 而不关联指标元数据。

## 5. 查询与穿透流程
1. 设计态保存图表配置时校验：
   - 对每个指标（含计算字段）检查当前机构是否存在对应指标元数据。
   - 不存在则保存失败并返回可读提示。
2. 查看态点击图表时：
   - 前端携带 `metricCode/metricName` 与图表上下文请求“指标解析”。
   - 后端按当前登录机构解析唯一 `metricId`。
   - 解析成功后跳转穿透页，继续使用现有 `/bi/drill/config/{metricId}` 与 `/bi/drill/query`。
3. 解析失败：
   - 返回“当前机构未注册该指标”的明确错误，不回退到临时计算字段穿透。

## 6. 接口建议
- 新增解析接口（推荐）：`POST /bi/metadata/resolve`
  - 入参：`metricCode`（必填）/`metricName`（可选）/`datasetId`（可选）
  - 上下文：从登录态获取机构ID，不接受前端伪造机构ID
  - 出参：`metricId`、`metricCode`、`metricName`
- 安全要求：
  - 仅返回当前机构可访问指标
  - 记录审计日志（用户、机构、请求标识、解析结果）

## 7. 错误处理与提示
- 设计态：`计算字段未注册为本机构指标元数据，无法启用穿透`。
- 运行态：`当前机构未注册该指标，请联系本机构管理员维护指标元数据`。
- 冲突态（同机构多条命中）：返回“指标标识不唯一”，要求治理修复。

## 8. 验收标准
1. 机构A/B可使用相同 `metricCode`，互不影响。
2. 机构A点击图表时只解析到机构A的 `metricId`。
3. 未注册指标的计算字段无法穿透，且提示明确。
4. 注册后同图表点击可正常进入穿透页。
5. 现有钻取接口仍基于 `metricId` 工作，无回归。

## 9. 风险与迁移
- 风险：历史图表只存表达式未存 `metricCode`，需要补齐映射。
- 迁移建议：提供批量检查脚本，扫描图表配置中未绑定机构指标元数据的计算字段并出清单。
