# 2026-03-09 BI穿透明细设计

## 1. 目标
在仪表板图表上点击数据点后，一键跳转到独立明细页，自动带入图表查询条件与指标口径条件，并支持用户在明细页动态追加筛选规则进行查询。

## 2. 已确认决策
- 入口方式：独立路由页面（非弹窗）
- 明细配置粒度：按指标统一配置（metricId）
- 折线图多指标：点击哪条线传哪条线的 metricId
- 明细规则：支持多条件组，组间可选 AND/OR
- 规则持久化：仅本次查询有效，不保存
- 冲突优先级：用户规则覆盖已带入条件
- 区块A已带入条件：
  - 包含图表查询条件带入
  - 包含计算字段（指标表达式）解析出的口径条件
  - 口径条件默认只读，但支持“转为可编辑条件”
- 表达式解析失败：不阻断，提示“口径条件解析失败，可手动补充”

## 3. 页面设计
### 3.1 路由
- 新增：`/bi/dashboard/drill/:dashboardId`
- 路由参数建议：
  - `metricId`
  - `componentId`
  - `pointTime`（可选）
  - `pointDimension`（可选）
  - `querySnapshot`（JSON或结构化对象）

### 3.2 区块A：已带入条件（只读）
- 分组展示：
  - 图表查询条件（机构、日期、查询组件值）
  - 指标口径条件（从表达式自动解析）
- 每条口径条件提供按钮：`转为可编辑条件`
- 解析失败提示：
  - `口径条件解析失败，可手动补充筛选规则`

### 3.3 区块B：指标筛选规则（可编辑）
- 支持多个条件组
- 组间关系：AND/OR
- 组内条件行结构：
  - 字段（中文显示，提交英文编码）
  - 操作符（中文显示）
  - 条件值输入框
- 常见操作符：等于、不等于、包含、大于、小于、介于、为空、非空
- 支持新增/删除条件组、条件行

### 3.4 区块C：明细结果
- 表格 + 分页
- 查询/重置
- 响应规则修改实时刷新（或按钮触发刷新）

## 4. 数据与配置模型
### 4.1 指标级明细配置（新增）
- `metricId`
- `detailDatasetId` 或 `detailSqlTemplate`
- `paramMapping`（例如 `sjbsjgid <- orgId`, `load_date <- pointTime`）
- `fieldDict`：字段中英文映射、数据类型、可用操作符

### 4.2 请求模型（明细查询）
- `metricId`
- `dashboardId`
- `componentId`
- `inheritedConditions`（区块A汇总）
- `ruleGroups`（区块B条件组）
- `pageNum/pageSize`

### 4.3 合并规则
1. 先注入带入条件（图表条件 + 口径条件）
2. 再应用用户条件组
3. 若同字段冲突，用户条件覆盖带入条件

## 5. 解析与容错
### 5.1 指标表达式解析
- 针对 `CASE WHEN ... THEN ...`、`IN (...)`、比较表达式等常见模式提取条件
- 输出结构化条件列表供区块A展示与“转为可编辑”复用

### 5.2 失败降级
- 解析失败不阻断查询
- 保留 metricId + 图表查询条件继续查询
- UI明确告知用户可在区块B手动补充

## 6. 安全与约束
- 字段白名单校验（仅 `fieldDict` 中允许字段可查询）
- 操作符白名单校验（按字段类型限制）
- SQL参数化执行，禁止拼接原始输入
- 记录审计日志：metricId、带入条件、最终条件组、查询耗时

## 7. 与现有能力衔接
- 复用已有图表点击事件：`metric-click` / `multi-metric-click`
- 在点击处理里组装穿透参数并路由跳转
- 保持现有查询组件与条件映射逻辑不变

## 8. 验收标准
- 单指标仪表盘：点击后进入明细页，机构/日期/指标自动带入
- 多指标折线：点击A线进入A指标明细，点击B线进入B指标明细
- 区块A可展示口径条件，支持“转为可编辑条件”
- 区块B支持多条件组 AND/OR 动态查询
- 冲突字段以用户规则为准
- 解析失败时可继续查询并有明确提示
